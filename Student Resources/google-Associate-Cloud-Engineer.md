# Google Associate Cloud Engineer Certification Prep

## Exam Overview
- **Duration:** 2 hours
- **Format:** 50-60 multiple choice and multiple select questions
- **Cost:** $125 USD
- **Language:** English, Japanese, Spanish, Portuguese, French, German, Indonesian
- **Validity:** 3 years

## Exam Domains

### 1. Setting up a Cloud Solution Environment (~17.5%)
- Setting up cloud projects and accounts
- Managing billing configuration
- Installing and configuring the Cloud SDK

### 2. Planning and Configuring a Cloud Solution (~17.5%)
- Planning and estimating GCP product use
- Planning and configuring compute resources
- Planning and configuring data storage options
- Planning and configuring network resources

### 3. Deploying and Implementing a Cloud Solution (~25%)
- Deploying and implementing Compute Engine resources
- Deploying and implementing Google Kubernetes Engine resources
- Deploying and implementing Cloud Run and Cloud Functions resources
- Deploying and implementing data solutions
- Deploying and implementing networking resources
- Deploying a solution using Cloud Marketplace
- Implementing resources via infrastructure as code

### 4. Ensuring Successful Operation of a Cloud Solution (~20%)
- Managing Compute Engine resources
- Managing Google Kubernetes Engine resources
- Managing Cloud Run resources
- Managing storage and database solutions
- Managing networking resources
- Monitoring and logging

### 5. Configuring Access and Security (~20%)
- Managing Identity and Access Management (IAM)
- Managing service accounts
- Viewing audit logs

## Key Services to Master

### Compute
- **Compute Engine:** VMs, instance groups, load balancers
- **Google Kubernetes Engine (GKE):** Clusters, pods, deployments
- **Cloud Run:** Serverless containers
- **Cloud Functions:** Event-driven serverless functions
- **App Engine:** Managed application platform

### Storage
- **Cloud Storage:** Object storage (Standard, Nearline, Coldline, Archive)
- **Persistent Disk:** Block storage for VMs
- **Cloud SQL:** Managed relational databases (MySQL, PostgreSQL, SQL Server)
- **Cloud Spanner:** Horizontally scalable relational database
- **Cloud Bigtable:** NoSQL wide-column database
- **Firestore:** NoSQL document database
- **Cloud Filestore:** Managed file storage

### Networking
- **VPC:** Virtual Private Cloud
- **Cloud Load Balancing:** HTTP(S), TCP/SSL, UDP
- **Cloud CDN:** Content delivery network
- **Cloud DNS:** Domain name system
- **Cloud VPN:** Secure network connections
- **Cloud Interconnect:** Dedicated connectivity
- **Firewall Rules:** Network security

### Management Tools
- **Cloud Console:** Web-based UI
- **Cloud SDK (gcloud):** Command-line interface
- **Cloud Shell:** Browser-based shell
- **Cloud Monitoring:** Performance monitoring
- **Cloud Logging:** Log management
- **Cloud Deployment Manager:** Infrastructure as code

## Tips
- Focus on hands-on experience with gcloud CLI
- Understand IAM roles and permissions thoroughly
- Practice creating and managing resources across all core services
- Know when to use each storage and compute option
- Understand basic networking concepts and VPC configuration
- Review billing and cost management best practices
