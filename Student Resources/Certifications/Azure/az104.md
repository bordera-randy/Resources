## üß© Core Topics & Links

**Deep dive into each exam domain with key concepts and resources.**

### Manage Azure identities & governance (20‚Äì25%)

**Key concepts:**
- **Entra ID (Azure AD):** Users, groups, licenses, external users, self-service password reset (SSPR)
- **RBAC:** Scopes, role assignments, interpreting access permissions
- **Management groups:** Subscriptions, resource groups, tags
- **Governance:** Azure Policy, resource locks
- **Cost controls:** Budgets, alerts, Azure Advisor recommendations

**Resources:**
- [Microsoft Entra ID Documentation](https://learn.microsoft.com/en-us/entra/identity/)
- [Azure RBAC Documentation](https://learn.microsoft.com/en-us/azure/role-based-access-control/)
- [Azure Policy Documentation](https://learn.microsoft.com/en-us/azure/governance/policy/)

---

### Implement & manage storage (15‚Äì20%)

**Key concepts:**
- **Security:** Storage firewalls, VNet integration, SAS tokens, stored access policies, identity-based access for Azure Files
- **Storage accounts:** Redundancy options, object replication, encryption, AzCopy, Storage Explorer
- **Azure Files & Blob:** Tiers (hot/cool/archive), soft delete, snapshots, lifecycle management, blob versioning

**Resources:**
- [Azure Storage Documentation](https://learn.microsoft.com/en-us/azure/storage/)
- [Azure Blob Storage Documentation](https://learn.microsoft.com/en-us/azure/storage/blobs/)
- [Azure Files Documentation](https://learn.microsoft.com/en-us/azure/storage/files/)

---

### Deploy & manage compute (20‚Äì25%)

**Key concepts:**
- **ARM templates / Bicep:** Interpret, modify, deploy; export and convert templates
- **Virtual Machines:** Create, sizing, disk management, encryption, migration (RG/sub/region), availability zones, availability sets, VM Scale Sets (VMSS)
- **Containers:** Azure Container Registry (ACR), Azure Container Instances (ACI), Azure Container Apps; sizing and scaling
- **App Service:** App Service plans, scaling, TLS/certificates, custom DNS, backups, networking, deployment slots

**Resources:**
- [Azure Virtual Machines Documentation](https://learn.microsoft.com/en-us/azure/virtual-machines/)
- [Azure App Service Documentation](https://learn.microsoft.com/en-us/azure/app-service/)
- [Bicep Documentation](https://learn.microsoft.com/en-us/azure/azure-resource-manager/bicep/)
- [Azure Container Services](https://learn.microsoft.com/en-us/azure/container-instances/)

---

### Implement & manage virtual networking (15‚Äì20%)

**Key concepts:**
- **Networking basics:** VNets, subnets, VNet peering, public IPs, user-defined routes (UDRs); troubleshoot connectivity
- **Secure access:** Network Security Groups (NSG), Application Security Groups (ASG), effective security rules, Azure Bastion, service endpoints, Private Endpoints
- **Name resolution & load balancing:** Azure DNS (private/public), internal/public load balancers; troubleshoot load balancing

**Resources:**
- [Azure Virtual Network Documentation](https://learn.microsoft.com/en-us/azure/virtual-network/)
- [Azure Load Balancer Documentation](https://learn.microsoft.com/en-us/azure/load-balancer/)
- [Azure Bastion Documentation](https://learn.microsoft.com/en-us/azure/bastion/)
- [Azure DNS Documentation](https://learn.microsoft.com/en-us/azure/dns/)

---

### Monitor & maintain Azure resources (10‚Äì15%)

**Key concepts:**
- **Azure Monitor:** Metrics, logs, KQL queries, alerts, action groups, alert processing rules
- **Insights:** VM insights, storage insights, network insights; Network Watcher, Connection Monitor
- **Backup & DR:** Azure Backup vaults, backup policies, restore operations; Azure Site Recovery for regional failover; backup reporting and alerts

**Resources:**
- [Azure Monitor Documentation](https://learn.microsoft.com/en-us/azure/azure-monitor/)
- [Azure Backup Documentation](https://learn.microsoft.com/en-us/azure/backup/)
- [Azure Site Recovery Documentation](https://learn.microsoft.com/en-us/azure/site-recovery/)
- [Network Watcher Documentation](https://learn.microsoft.com/en-us/azure/network-watcher/)

## üß† Tips & Study Strategy

**Maximize your success with these proven strategies:**

1. **Map your prep to exam domains:** Use the five skill areas above as your study checklist.
2. **Do focused, hands-on labs:** For each domain, practice small labs (e.g., NSG + Private Endpoint + Bastion; Bicep deployment of VM + Disk + Policy).
3. **Stay current:** Review the [exam change log](https://learn.microsoft.com/en-us/certifications/exams/az-104) to avoid studying retired topics.
4. **Master KQL basics:** Practice Kusto Query Language for Azure Monitor queries and alert creation.
5. **Take timed practice tests:** Simulate exam conditions with 90‚Äì100 minute practice exams. Review every missed question and find the official Microsoft documentation.
6. **Use the sandbox:** Microsoft Learn provides free Azure sandbox environments for hands-on practice.
7. **Join study groups:** Engage with communities on Reddit, Discord, or Microsoft Tech Community for tips and support.

**Recommended study timeline:**
- **Beginner:** 8‚Äì12 weeks (10‚Äì15 hours/week)
- **Intermediate:** 4‚Äì6 weeks (10‚Äì15 hours/week)
