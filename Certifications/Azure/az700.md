### Design & implement core networking infrastructure (25‚Äì30%)

**Key concepts:**
- **IP addressing & segmentation:** CIDR notation, subnets, private IP ranges, planning
- **Name resolution:** Azure DNS (public/private zones), private resolvers, conditional forwarding
- **VNet connectivity:** Virtual networks, peering (local/global), user-defined routes (UDRs)
- **Network address translation:** NAT gateway, outbound connectivity patterns
- **Routing:** Route tables, route priorities, Azure Route Server
- **Monitoring:** Network Watcher, Azure Monitor Network Insights, diagnostics

## üéØ Tips & Study Strategy

**Proven strategies to master AZ-700:**

1. **Map domains to study time:**
   - Allocate study time proportionally: 25-30% on core networking, 20-25% on connectivity, etc.
   - Create a study checklist for each domain

2. **Hands-on practice is essential:**
   - Build a complete VNet environment: subnets, peering, route tables, NAT gateway
   - Configure VPN Gateway, ExpressRoute, or Virtual WAN connectivity
   - Deploy Load Balancer, Application Gateway, and Front Door with real traffic patterns
   - Practice tearing down and rebuilding environments

3. **Master hybrid connectivity scenarios:**
   - Know when to use VPN Gateway vs ExpressRoute vs Virtual WAN
   - Understand failover, redundancy, and high-availability patterns
   - Practice site-to-site VPN and point-to-site VPN configurations

4. **Deep dive into application delivery:**
   - Understand Load Balancer (Layer 4) vs Application Gateway (Layer 7) vs Front Door
   - Know SKUs, features, SSL/TLS termination, health probes
   - Practice routing rules, WAF policies, and caching strategies

5. **Network security is critical:**
   - Master NSGs, ASGs, Azure Firewall, and WAF rules
   - Understand hub-and-spoke architecture and zero-trust segmentation
   - Practice DDoS protection configuration

6. **Private access and DNS:**
   - Understand Private Link vs Service Endpoints
   - Practice private endpoint DNS configuration
   - Know private DNS zones and conditional forwarding

7. **Monitoring and diagnostics:**
   - Use Network Watcher for connectivity troubleshooting
   - Understand Azure Monitor Network Insights for performance analysis
   - Know flow logs and traffic analytics

8. **Take timed practice exams:**
   - Simulate real exam conditions with 100‚Äì120 minute exams
   - Practice time management: ~1.5-2 minutes per question
   - Review every incorrect answer against official documentation

**Recommended study timeline:**
- **Networking beginners:** 10‚Äì14 weeks (10‚Äì15 hours/week)
- **Some Azure experience:** 6‚Äì8 weeks (10‚Äì15 hours/week)
- **Network administrators/architects:** 3‚Äì4 weeks (focused review and practice)
